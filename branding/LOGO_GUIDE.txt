LOGO SPECIFICATION
==================

Drop a logo file into this folder (branding/), restart the app, and it appears
in both the console UI and the PDF reports.

FILE NAME
  Must be named logo.png or logo.jpg (lowercase).
  The app looks for png first, then jpg. Just drop it in — no config to edit.

SUPPORTED FORMATS
  YES  PNG  (recommended — supports transparency, looks best on the dark header)
  YES  JPG / JPEG
  NO   SVG  (PDF reports cannot embed vector images; export to PNG instead)

SIZE REQUIREMENTS
  - Aspect ratio: a wide horizontal logo bar (about 3:1 to 5:1) works best.
  - Recommended resolution: 128-256 px tall, width to match
    (e.g. 512x128, 640x160). Larger is fine (auto-scaled) but unnecessary.
  - Minimum usable: at least 64 px tall, or it looks blurry in the PDF.
  - File size: keep under 1 MB. A PNG logo is usually only a few dozen KB.

WHERE IT SHOWS
  - Console top-left: auto-scaled to 32 px tall, width proportional.
  - PDF report: top of the cover page (~0.9 in tall) + every page header (0.3 in).
  - A transparent PNG looks best on the dark header; a JPG's white background
    will show as a white box.

TO APPLY
  After adding the logo, close the desktop app window and reopen it
  (the app reads this folder at startup).
